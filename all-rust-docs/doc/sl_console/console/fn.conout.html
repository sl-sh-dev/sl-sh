Lock and return write side of the tty/console for the application.

This provides a Write object that is connected to /dev/tty (unix) or the console (windows). This will not write to standard output (if it is not the tty/console), but rather write to the TTY or console device. In other words, if you pipe standard output to another process things you write to conout() will not go into the pipe but will go to the terminal. This will always return the the locked output console, will panic if it does not exit. Always call coninit() once and do not call conout() if it returns an error.
